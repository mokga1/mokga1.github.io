# Godius 캐릭터 육성 평가기 — 설계 문서

날짜: 2026-07-10
대상 게임: 가디우스 (Steam: Godius Eternal War)
데이터 출처: 가디우스 공식 홈페이지 가이드 (godius.co.kr — 기능 및 파라메터, 캐릭터 직업, 캐릭터 생성하기 페이지) + 사용자 제공 기능 성공률

## 목적

사용자가 캐릭터의 직업·레벨·현재 파라메터·기능(스킬) 레벨을 입력하면,
레벨업으로 획득한 [힘의 구슬] 총량 대비 실제 소비 내역을 계산하여
육성이 효율적으로 되었는지(기능 실패로 구슬을 얼마나 날렸는지) 등급과 상세 분석으로 평가한다.

## 형태

- 단일 `index.html` 파일. 서버 불필요, 브라우저에서 바로 열림.
- 게임 데이터(직업, 초기 스탯, 구슬 획득표, 비용표, 성공률)는 파일 상단 JS 상수 객체로 내장하여 수정이 쉽도록 한다.
- UI 언어: 한국어.

### HP/MP/SP 성장 진단 (2026-07-11 추가)

- 생성 시 초기치: HP = 초기 VIT×2, MP = 초기 MEN×2, SP = 초기 STR×2 (공식).
- 레벨업 1회당 상승 범위 (커뮤니티 공략, 주직업 기준 — 보조직업 영향은 공식 미공개로 미반영):

| | 전사 | 검투사 | 도둑 | 마법사 | 성직자 |
|---|---|---|---|---|---|
| HP | 2~3 | 2~3 | 1~3 | 1~2 | 1~3 |
| MP | 1~2 | 1~2 | 1~3 | 2~3 | 1~3 |
| SP | 1~3 | 1~3 | 1~3 | 1~2 | 1~3 |

- 사용자가 현재 HP/MP/SP를 선택 입력하면 기대 범위(초기치 + 레벨업×범위) 내 위치(백분위)를 계산해
  평균 이상/평균 수준/평균 이하 판정. 범위 밖이면 입력 확인 경고(장비 보정 의심 등).
- 초기 스탯은 autoAllocate(핵심 스탯 우선 15까지 배분) 추정치 사용. 등급(구슬 효율)에는 영향 없음 — 별도 진단 표.

## 계산 로직

1. **획득 구슬** `earned` = 레벨 2부터 현재 레벨까지 획득표 누적 합.
2. **파라메터 소비** `paramCost`:
   - 각 스탯의 (직업 기본값 → 현재값) 상승 비용 합을 구한 뒤, 보너스 포인트만큼 절약분을 뺀다.
   - 초기 배분은 사용자에게 **유리한 쪽으로 자동 추정**: 15 이하 구간 포인트는 모두 1개짜리이므로,
     배치 가능한 보너스 = min(보너스포인트, Σ max(0, min(15, 현재값) − 기본값)). 절약 = 배치 가능한 보너스 × 1개.
3. **기능 소비**:
   - 이론 최소 `skillMin` = Σ (현재 기능레벨 − 1) × 1개.
   - 평균 기대치 `skillExpected` = Σ (레벨 2→현재까지 각 단계의 1/성공률).
   - 실제 소비 `skillActual` = earned − paramCost − 미사용 구슬.
4. **효율 지수** = skillExpected / skillActual (기능을 하나도 안 올렸으면 별도 처리).
   - 1.0 = 평균적인 유저, 1보다 크면 평균보다 운이 좋았거나 효율적.

### 등급 기준 (효율 지수)

| 등급 | 기준 |
|---|---|
| S | ≥ 1.3 |
| A | ≥ 1.1 |
| B | ≥ 0.9 |
| C | ≥ 0.7 |
| D | ≥ 0.5 |
| F | < 0.5 |

기능 투자가 전혀 없는 경우(기능 전부 레벨 1): 미사용 구슬로 수지가 맞으면 "평가 유보(구슬 비축형)", 모자라면 F.

## 입력 검증

- 현재 파라메터 ≥ 직업 기본값, ≤ 30.
- Σ min(15, 현재값) ≥ 72 (초기 배분 72가 불가능한 입력 차단).
- skillActual < skillMin 이면 "입력이 모순됨(구슬 수지 불일치)" 경고와 함께 어느 입력을 다시 확인할지 안내.
- skillActual < 0 이면 획득량보다 많이 소비한 불가능한 캐릭터 → 오류 안내.

## UI 구성 (단일 페이지, 4단계 + 결과)

1. 주직업 / 보조직업 선택 (카드형 버튼, 제약 자동 반영)
2. 레벨 입력 (2~99)
3. 현재 파라메터 6종 입력 (기본값 프리필, 검증 실시간 표시)
4. 보유 기능 레벨 입력 (직업 조합에 따라 목록 자동 생성, 각 1~30) + 미사용 힘의구슬 (선택, 기본 0)
5. [평가하기] → 결과:
   - S~F 등급 카드 (큰 표시)
   - 구슬 수지표: 획득 / 파라메터 소비 / 기능 이론최소 / 기능 평균 기대치 / 실제 기능 소비 / 낭비 추정
   - 운 게이지: "평균 유저 대비 구슬 N개 절약(또는 초과 소모)"
   - 스탯 진단 코멘트: 공식 Tip 기반 — 전사 계열은 STR·VIT 15 권장, 마법사 계열은 VIT·MEN·INT 15 권장. 주직업 핵심 스탯 저투자/비핵심 스탯 과투자 지적.

## 테스트

- 계산 함수(획득/파라메터/기능 기대치/등급)는 순수 함수로 분리하고, 페이지 내 `?test` 쿼리 또는 콘솔용 self-test 함수로 케이스 검증한다.
  - 예: 레벨 10 → earned 27, 레벨 30 → 67, 레벨 50 → 147.
  - 파라메터 6→30 비용 = 9×1 + 5×3 + 5×4 + 5×5 = 69.
  - 초기 배분 절약 로직, 검증 오류 케이스.

## 가정 (데이터 불확실 지점, JS 상수로 수정 가능하게 표기)

- 기능 시작 레벨 1.
- 기능 성공률은 사용자 제공 근사치.
- 검투사 기능 목록 일부는 가이드 이미지 잘림으로 추정.
- 보조직업 제약: 검투사 → 성직자·연금술사 불가 외에는 전 조합 허용, 주=보조 동일 직업 불가.
- 신규 기능 습득(배우기) 비용은 0으로 가정 (직업 기능은 생성 시 보유).
